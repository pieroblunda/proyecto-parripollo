1. El Origen: TPS y el Nacimiento de "Lean"

En el centro de todo está el Toyota Production System (TPS). Fue el "paciente cero". Desarrollado por Taiichi Ohno, su objetivo era eliminar el desperdicio (muda) y maximizar el valor. Cuando los académicos occidentales estudiaron los milagros de Toyota en los años 80, bautizaron esta filosofía como Lean Manufacturing. 

2. El Duelo de Motores: Secuencial vs. Iterativo
Dentro del gran universo de las Metodologías de Desarrollo de Software, existen dos formas fundamentales de mover el motor del progreso:

Desarrollo Secuencial (Waterfall): Es el "viejo orden". Se basa en etapas rígidas: Requisitos → Diseño → Implementación → Pruebas. No puedes volver atrás. Es como construir un edificio; si decides que quieres un sótano cuando ya vas por el quinto piso, tienes un problema grave.

Desarrollo Iterativo: Es el precursor biológico de Agile. En lugar de una gran entrega final, el proyecto se divide en ciclos pequeños. Cada ciclo es una versión miniatura del proyecto que se va refinando.

Concepto	Enfoque	Filosofía
Secuencial	Lineal y rígido	"Hazlo bien a la primera"
Iterativo	Cíclico y fluido	"Aprende y mejora en cada vuelta"

3. La Gran Convergencia: Agile Software Development
Aquí es donde todo se une. El Agile Software Development es el movimiento que toma la filosofía Lean (eliminar lo que no aporta valor) y la mecánica del Desarrollo Iterativo (ciclos cortos de feedback) para crear un nuevo estándar.

La jerarquía relacional queda así:

Metodologías de Desarrollo de Software: El contenedor global que incluye todas las formas de crear programas.

Lean Manufacturing / TPS: La base filosófica. Aporta los valores de fluidez, respeto por las personas y eliminación de tareas inútiles.

Agile: Es el "paraguas" bajo el cual viven marcos como Scrum o Kanban. Agile usa los principios Lean para gestionar el trabajo.

Desarrollo Iterativo: Es la herramienta técnica que usa Agile para funcionar. Sin iteración, no hay agilidad.

Desarrollo Secuencial: Es el contraste, la metodología que Agile y Lean intentan superar para evitar proyectos obsoletos antes de nacer.

En resumen: TPS nos enseñó a eliminar el desperdicio; el Desarrollo Iterativo nos dio el ritmo; y Agile unió ambos para rescatarnos de la rigidez del Desarrollo Secuencial.
