# Sensibilidad por los requeriemientos no funcionales

`Other skills`

La gente se concentra solo por los requerimientos funcionales e ignorar los atributos no funcionales hacen un software de mala calidad. El cliente menciona solo los requisitos funcionales y da por hecho que los requisitos no funcionales van a estar. Entonces los requisitos no funcionales muchas veces no son presupuestados. El problema se agrava aun mas cuando intentan presupuestarse, y se reserva un "30%" simbolico. Lo que pasa con esto ultimo es que el desarrollo de los requisitos funcionales siguen la ley de parkinson, y nunca hay tiempo para mejorar requisitos no funcionales (como debito tecnico, velocidad, refactoring, ecc)

Lo loco de esto es que se llaman "Requerimientos" pero el cliente no los requiere.
